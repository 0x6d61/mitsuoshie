# Mitsuoshie（ミツオシエ）設計書 v2

## コンセプト

Windows向けハニートークン検知ツール。
インストールするだけで罠ファイルを自動配置し、アクセスされたら即通知する。
誤検知ほぼゼロ。機能は最小限。触られたら黒確定。

**「蜜の場所を教え、侵入者を罠に導く鳥 — Honeyguide」**

---

## なぜハニートークンか — LLMエージェント時代の検知

従来のマルウェアは決まったパターンでファイルを漁る。Infostealerは
.aws/credentials, .ssh/id_rsa, wallet.dat を機械的に列挙する。
これに対してシグネチャ検知は追いかけっこになる。

LLMエージェントが攻撃に使われる時代では、エージェントは
「クレデンシャルを探して」という自然言語の指示に従い、
人間のようにファイルシステムを探索する。
ルールに「ハニートークンを避けろ」と書いてなければ普通にアクセスする。

ハニートークンの強みは：
- 攻撃手法に依存しない（シグネチャ不要）
- LLMエージェントもInfostealerも同じ罠にかかる
- 誤検知がほぼゼロ（正規アプリは罠ファイルにアクセスしない）
- 攻撃者が罠を避けるには、罠の存在を事前に知る必要がある

---

## 設計原則

1. **やることは3つだけ**: 罠を撒く、監視する、通知する
2. **ブロックしない**: 検知と通知のみ
3. **誤検知ゼロを目指す**: 正規アプリが絶対にアクセスしないファイルだけを罠にする
4. **インストールしたら放置**: 設定不要、メンテ不要
5. **ログは標準形式**: Windows Event Log + Sysmon互換JSONで出力。保存・分析は外部ツールの仕事
6. **ハッシュ比較はデフォルト**: 罠ファイル配置時にSHA256を記録し、定期チェック時にハッシュ比較。改ざんされていたら即アラート
7. **監視はカーネルに任せる**: NTFS監査（SACL）でWindowsカーネルがアクセスを記録。自前の監視エンジンは持たない

## 旧設計（FileSystemWatcher）との比較

| 項目 | 旧: FileSystemWatcher | 新: SACL + Event Log |
|------|----------------------|---------------------|
| 読み取り検知 | ✗ 不可能 | ✓ ReadData で検知 |
| プロセス特定 | ✗ 別途P/Invokeが必要、タイムラグあり | ✓ Event 4663にPID/プロセス名が含まれる |
| 取りこぼし | あり（短命プロセス） | なし（カーネルが記録） |
| 監視エンジン | 自前実装 | Windowsカーネル（枯れてる） |
| 管理者権限 | 不要 | 初回SACL設定時のみ必要 |
| 依存 | .NET FileSystemWatcher | Windows監査ポリシー |

## SACL設定

### 監査ポリシーの有効化

```csharp
// 初回セットアップ時に「オブジェクトアクセスの監査」を有効化
// ローカルセキュリティポリシー相当の操作

// 方法: auditpol コマンド
// auditpol /set /subcategory:"File System" /success:enable /failure:enable

public static void EnableFileSystemAuditing()
{
    var psi = new ProcessStartInfo
    {
        FileName = "auditpol",
        Arguments = "/set /subcategory:\"File System\" /success:enable",
        Verb = "runas",  // 管理者権限で実行
        UseShellExecute = true,
        CreateNoWindow = true
    };
    Process.Start(psi)?.WaitForExit();
}
```

### 罠ファイルへのSACL設定

```csharp
// 各罠ファイルにNTFS監査エントリを追加
// Everyone に対する ReadData, WriteData, Delete を監査

public static void SetHoneyTokenAudit(string filePath)
{
    var fileInfo = new FileInfo(filePath);
    var security = fileInfo.GetAccessControl();

    // Everyone の ReadData, WriteData, Delete, AppendData を監査
    var auditRule = new FileSystemAuditRule(
        identity: new SecurityIdentifier(WellKnownSidType.WorldSid, null),
        fileSystemRights: FileSystemRights.ReadData
                        | FileSystemRights.WriteData
                        | FileSystemRights.Delete
                        | FileSystemRights.AppendData,
        flags: AuditFlags.Success
    );

    security.AddAuditRule(auditRule);
    fileInfo.SetAccessControl(security);
}
```

## explorer.exe の扱い

```
ReadAttributes (0x80) = フォルダ表示 → 除外（正常なUI操作）
ReadData (0x1)        = ファイル内容読み取り → アラート対象

explorer.exe のアクセスマスクで区別する:
  - ReadAttributes のみ → 除外（フォルダをエクスプローラーで開いただけ）
  - ReadData を含む   → アラート（ファイルを開いた/プレビューした）

これにより explorer.exe を一律除外する必要がなくなり、
攻撃者がエクスプローラー経由でファイル内容を読んだ場合も検知できる。
```

## ハニートークン配置設計

### 配置するダミーファイル

攻撃者やInfostealerが狙うファイルパターンを模倣。
正規アプリケーションが絶対にアクセスしないパスに配置する。

```
配置先と罠ファイル:

C:\Users\<user>\
├── .aws\
│   └── credentials.bak          # AWS クレデンシャルのバックアップを装う
├── .ssh\
│   └── id_rsa.old               # SSH秘密鍵の旧版を装う
├── Documents\.secure\            # 隠しフォルダ
│   ├── passwords.xlsx            # パスワード一覧を装う
│   └── wallet_backup.dat         # 暗号通貨ウォレットを装う
├── Desktop\.confidential\        # 隠しフォルダ
│   └── 重要_機密情報.docx         # 機密ファイルを装う
├── .config\
│   └── .env.production           # 本番環境の環境変数を装う
├── AppData\Local\
│   └── Google\Chrome\User Data\
│       └── Login Data.bak        # ブラウザ保存パスワードのバックアップを装う
└── AppData\Roaming\
    └── Bitcoin\
        └── wallet.dat.bak        # Bitcoinウォレットのバックアップを装う

※ Documents\.secure\ と Desktop\.confidential\ は隠しフォルダ属性を付与。
  ユーザーの目には見えないが、攻撃者のファイル列挙スクリプトは
  隠しフォルダも走査するため罠として機能する。
```

### ダミーファイルの中身

```
全ファイル共通で以下のフォーマット:
- ファイルタイプに応じた本物っぽいヘッダー
- 中身はダミーデータ（実際のクレデンシャルは含まない）
- 末尾に不可視のMitsuoshie識別子を埋め込む（改ざん検知用）
- ダミーキーはランダム生成し、既知のサンプルキーは使用しない

例: credentials.bak
---
[default]
aws_access_key_id = AKIA<ランダム16文字>
aws_secret_access_key = <ランダム40文字>
region = ap-northeast-1
---

例: .env.production
---
DATABASE_URL=postgresql://admin:<ランダム>@db.internal.local:5432/prod
API_SECRET=sk_live_<ランダム32文字>
STRIPE_KEY=pk_live_<ランダム32文字>
---

例: id_rsa.old
---
-----BEGIN OPENSSH PRIVATE KEY-----
<ランダムBase64データ>
-----END OPENSSH PRIVATE KEY-----
---
※ 全て無効なダミーデータ。有効なキーは一切含まない。
```

### 配置ルール

```
1. 既存のファイルは絶対に上書きしない
   → パスにファイルが既に存在する場合はスキップ
2. 配置先ディレクトリが存在しない場合は作成する
   → .aws, .ssh, .config, Bitcoin 等のディレクトリも含めて自動作成
   → 攻撃者はディレクトリの存在を見て「このマシンにクレデンシャルがある」と判断する
3. Desktop\.confidential\, Documents\.secure\ は隠しフォルダ属性を付与
   → File.SetAttributes(dirPath, FileAttributes.Hidden)
   → ユーザーの普段使いに干渉しない
   → 攻撃者のDir /A:H やGet-ChildItem -Force は隠しフォルダも列挙する
4. 各罠ファイルにSACL（監査ACL）を設定
5. 配置したファイルの一覧とSHA256ハッシュを設定JSONに記録
6. ファイルの更新日時を過去の日付に設定（最近作られたファイルに見えないように）
7. ファイル属性は通常（Hidden にしない、攻撃者に見つけてもらう必要がある）
   ※ フォルダは隠す、ファイル自体は隠さない
8. ダミーキーはインストール時にランダム生成（既知サンプルキーは使わない）
```

## Security Event Log 購読

### イベント購読

```csharp
// Security Event Log の Event ID 4663 を購読
// 罠ファイルへのアクセスのみフィルタリング

public class SecurityEventSubscriber : IDisposable
{
    private EventLogWatcher _watcher;
    private readonly HashSet<string> _honeyPaths;

    public void Start()
    {
        // XPath クエリで Event ID 4663 のみ購読
        var query = new EventLogQuery(
            "Security",
            PathType.LogName,
            "*[System[EventID=4663]]"
        );

        _watcher = new EventLogWatcher(query);
        _watcher.EventRecordWritten += OnSecurityEvent;
        _watcher.Enabled = true;
    }

    private void OnSecurityEvent(object sender, EventRecordWrittenEventArgs e)
    {
        var record = e.EventRecord;

        // イベントからファイルパスを抽出
        var objectName = record.Properties[6]?.Value?.ToString(); // Object Name

        // 罠ファイルでなければ無視
        if (!_honeyPaths.Contains(objectName?.ToLowerInvariant())) return;

        // アクセスマスクを抽出
        var accessMask = record.Properties[9]?.Value?.ToString(); // Access Mask

        // ReadAttributes のみ (0x80) なら除外（フォルダ表示）
        if (accessMask == "0x80") return;

        // プロセス情報を抽出
        var processId = Convert.ToInt32(record.Properties[8]?.Value);  // Process ID
        var processName = record.Properties[11]?.Value?.ToString();     // Process Name

        // 安全プロセス除外
        if (IsSafeProcess(processName, accessMask)) return;

        // Mitsuoshie自身のアクセスは除外
        if (processId == Environment.ProcessId) return;

        // ハッシュ比較
        var currentHash = ComputeSHA256(objectName);
        var originalHash = _tokenStore.GetOriginalHash(objectName);
        var tampered = currentHash != originalHash;

        // アラート生成
        var alert = new MitsuoshieAlert
        {
            Timestamp = record.TimeCreated ?? DateTime.UtcNow,
            HoneyFile = objectName,
            HoneyType = _tokenStore.GetHoneyType(objectName),
            EventType = tampered ? "Tampered" : GetAccessType(accessMask),
            Tampered = tampered,
            OriginalHash = originalHash,
            CurrentHash = currentHash,
            AccessMask = accessMask,
            ProcessId = processId,
            ProcessName = Path.GetFileName(processName),
            ProcessPath = processName,
            User = record.Properties[1]?.Value?.ToString(),
            Severity = AlertSeverity.Critical
        };

        // 通知 + ログ出力
        OnAlertGenerated?.Invoke(alert);
    }
}
```

### 安全プロセスの除外（改善版）

```csharp
// プロセス名だけでなくアクセスマスクも考慮して判定

private bool IsSafeProcess(string processPath, string accessMask)
{
    var processName = Path.GetFileName(processPath)?.ToLower();

    // Windows Defender / Search Indexer は常に除外
    var alwaysExclude = new[]
    {
        "msmpeng.exe",            // Windows Defender
        "searchindexer.exe",      // Windows Search
        "searchprotocolhost.exe", // Windows Search
        "tiworker.exe",           // Windows Update
        "consent.exe"             // UAC
    };

    if (alwaysExclude.Contains(processName)) return true;

    // explorer.exe は ReadAttributes のみ除外
    // ReadData を含む場合はアラート対象
    if (processName == "explorer.exe")
    {
        return accessMask == "0x80"; // ReadAttributes のみ
    }

    return false;
}
```

## 初回セットアップフロー

```
1. インストーラー実行（管理者権限）

3. 罠ファイルをランダムキーで生成・配置
4. 各罠ファイルにSACL設定
5. SHA256ハッシュを計算して設定JSONに記録
6. Mitsuoshieサービスをスタートアップに登録
7. Security Event Log の購読開始
8. タスクトレイに常駐

所要時間: 数秒
ユーザー操作: インストーラーのNext連打のみ
```
